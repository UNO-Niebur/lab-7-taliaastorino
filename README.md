# Lab 7 - Program Design with Functions (Project Euler)

## Purpose
This lab focuses on **designing programs using functions**. You will practice:
- breaking large problems into smaller parts
- writing reusable helper functions
- composing functions together
- using incremental development to build correct solutions

Project Euler problems are used as the context for this work.

Detailed explanations, expectations, and examples are provided in **Canvas**.  
This README describes only what you need to do **inside this repository**.

---

## Files You Will Work With

### `NumberTests.py`
This file should contain reusable helper functions.

You must implement:
- `isPrime(p)` – returns `True` if `p` is prime, otherwise `False`

You may also add other helper functions as needed (for example: checking even numbers, generating sequences, etc.).

Include a docstring for each function explaining what it does.

---

### Project Euler Problem Files
Choose **2–3** Project Euler problems from the list provided in Canvas.

For each problem:
- create a separate Python file (e.g., `problem3.py`)
- use functions from `NumberTests.py` where appropriate
- include a short header comment with:
  - the problem number
  - a brief description of your approach
  - any runtime notes

Each problem file should be runnable on its own.

---

## Development Guidelines

- Use **incremental development**:
  - write a small piece
  - test it
  - then add the next piece
- Avoid writing one large function that does everything
- Reuse helper functions whenever possible
- Focus on clarity and correctness over clever tricks

---

## Testing

Before submitting:
- test `isPrime(p)` with small known values
- verify each Project Euler solution runs correctly
- confirm output matches the expected problem answer

Testing is part of the lab.

---

## Submission

To submit Lab 7:
1. Commit all required files to this repository
2. Sync your changes with GitHub
3. Submit the **repository link** on Canvas

---

## Reminder

This lab is about **how you structure and design your solution**, not just the final numeric answer.

Multiple correct approaches are expected.
